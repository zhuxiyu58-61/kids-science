# 知识乐园 · 体系化设计方案

> 面向：小学 2 年级（约 7–8 岁），语数外之外的知识拓展
> 目的：有趣 + 易懂地拓展知识面，不限学科（理化生 / 政史地 / 自然 / 艺术）
> 形态：网页互动课（可点可玩）· 平板触屏 · 只用 Claude Code 生产
> 节奏：**每天一课，单课投入 15–20 分钟**（贴合 7–8 岁真实专注时长）

---

## 一、为什么要"体系化"而不是零散做

零散做的问题：知识点之间没有联系，孩子学完就忘，家长也不知道下一课该学啥。
体系化 = **领域 → 单元 → 每日一课** 三层结构 + 一张能不断长大的知识地图。

```
知识乐园
├── 🔬 自然科学
│   ├── 单元1：水与天气   （水的三态 / 云和雨 / 冰和雪 / 彩虹）
│   ├── 单元2：植物世界   （种子旅行 / 一棵树的一年 / 花的秘密）
│   └── 单元3：力与运动   （磁铁 / 浮和沉 / 推和拉）
├── 🌍 人文·历史·地理
│   ├── 单元1：方向与地图 （东南西北 / 我们的中国 / 地球长啥样）
│   └── 单元2：古人生活   （很久以前 / 古代小朋友的一天）
├── 🎨 艺术·音乐
│   ├── 单元1：色彩魔法   （三原色 / 冷色暖色 / 名画里的故事）
│   └── 单元2：声音世界   （高音低音 / 认识乐器）
└── 🧠 理化生萌芽
    ├── 单元1：我的身体   （心跳呼吸 / 五种感觉）
    └── 单元2：小动物     （虫子变蝴蝶 / 谁吃谁）
```

> 每个"单元"3–6 课，每课就是"一天"。孩子沿着一条线走，知识有连贯感。

---

## 二、单课结构：15–20 分钟怎么安排（核心）

7–8 岁孩子单个活动专注约 15–20 分钟，所以一课 = 一段紧凑的"闯关旅程"，
由 **4 个环节**串成，每环节自成一个小互动，节奏"看→玩→比→得"：

| 环节              | 时长      | 做什么              | 教学作用    |
| --------------- | ------- | ---------------- | ------- |
| 🎬 **1. 开场**    | 1–2 min | 情境勾子/一句提问把娃"勾"进来 | 导入·激发好奇 |
| 🔍 **2. 探索**    | 7–8 min | 主互动，随便点随便玩、看现象变化 | 探究·建立直觉 |
| 🎮 **3. 闯关/动手** | 5–6 min | 拖拽分类 / 小游戏把知识用起来 | 应用·巩固   |
| ⭐ **4. 小结+徽章**  | 1–2 min | 回顾三句话 + 得一枚收集徽章  | 反馈·成就感  |

**合计约 15–20 分钟**，不拖沓。徽章可收集 → 每天一枚 → 形成"打卡"动力
（对标 [[egg-game]] 的收集主线）。

### 以《水的三个样子》为例，一课长这样：

1. 🎬 开场：小水滴"水滴滴"说"我会变身哦，你猜我能变成几个样子？"
2. 🔍 探索：**烧杯互动**（加热/冷却看冰↔水↔水蒸气）← 已完成
3. 🎮 闯关：把"冰棍🧊 / 一杯水🥤 / 冒气的水壶🫖"拖进"固态/液态/气态"三个筐
4. ⭐ 徽章：获得【水的魔法师】徽章，回地图

## 三、技术与生产方式

- 纯前端，无构建步骤：每课一个自包含 `index.html`（Canvas + 原生 JS），平板直接开、也能部署
- 统一模板：`_template/` 放单课 6 环节骨架 + 公共样式/徽章组件，新课复制即用
- 主页 `index.html` = 知识地图 + "今天这一课" + 徽章墙
- 目录：
  
  ```
  kids-science/
  ├── index.html            主页/知识地图
  ├── _template/            单课6环节模板（待建）
  ├── lessons/
  │   └── water/index.html  第一课（探索环节已成，其余环节待补）
  └── data/lessons.js       课程清单+进度（待建）
  ```

## 四、推进节奏

- **第一步**：把《水的三个样子》从"1 个探索环节"补成"完整 6 环节 45–60 分钟"，作为**样板课/模板**
- **第二步**：抽出 `_template`，之后每天照模板产 1 课
- **第三步**：主页接入徽章墙 + 进度打卡
